active_accounts <br>

**main_accounts** - аккаунты, которые приглашают другие аккаунты при инвайте.<br>
**temporary_ban_accounts** - аккаунты со спамблоком <br>

## Запуск приложения

Запуск приложения происходит через следующий файл: app/start.py

Перед созданием экземпляра класса с окном, проверяем есть ли интернет через запрос: <br>
response = requests.get("http://www.google.com", timeout=5) <br>
если его нет, то не запустим приложение и выведем соответсвующие сообщение. <br>
так же берём имя нашего приложения и ищём его в запущенных процессах
```
script_name = os.path.basename(__file__)  # Получаем имя текущего скрипта 
for process in psutil.process_iter(['pid', 'name']): 
   try:
     if script_name in process.info['name']: # Проверка по имени процесса
```

Если такой процесс уже запушен,
то выводим соответствующие сообщение пользователю и не разрешаем открыть приложение.

Если всё выше не произошло, то создаём экземпляр класса MainWindow

В MainWindow находится один элемент "stacked_widget" в который в дальнейшем помещается другие окна. <br>

Изначально создаём экземпляры класса (внутри MainWindow). <br>
Пример: `self.accounts_window = Window_accounts(self.switch_window)` <br>
После добавляем эти элементы и запускаем первым окном окно с аккаунтами. <br>

В MainWindow имеется одна функция "switch_window" через неё происходи переключение между вкладками. <br>
Принимаемый параметр: target_window: str - вкладка которую необходимо открыть.

## Проверка прокси (check_proxy.py)
Этот модуль предоставляет функции для проверки прокси-сервера SOCKS5.<br>

**Функция "check_proxy": <br>**
def check_proxy(proxy_host: str, proxy_port: int, username: str = None, password: str = None) -> str or bool: <br>

**Описание:**
Это основная функция через которую происходит проверка.
Проверяет доступность прокси-сервера, отправляя запрос к http://httpbin.org/ip с использованием заданного прокси.<br>
Если прокси работает, возвращает IP-адрес, полученный от сервера, иначе вернёт False

**Параметры:**
- proxy_host (str): Хост-прокси (IP-адрес или доменное имя).
- proxy_port (int): Порт прокси-сервера.
- username (str, optional): Имя пользователя для аутентификации на прокси, если требуется. По умолчанию None.
- password (str, optional): Пароль для аутентификации на прокси, если требуется. По умолчанию None.

**Возвращаемое значение:**
- str или bool: Возвращает IP-адрес (str), если прокси работает и ответ от сервера успешен (статус 200). Если прокси недоступен или произошла ошибка, возвращает False.


**Функция "_set_socks5_proxy": <br>**

**def _set_socks5_proxy(proxy_host: str, proxy_port: int, username: str = None, password: str = None):<br>**

**Описание:<br>**
Эта функция предназначена для использования только внутри этого файла. <br>
Настраивает прокси-сервер SOCKS5, перенаправляя все исходящие соединения через указанный прокси.<br>

**Параметры:<br>**
- proxy_host (str): Хост-прокси (IP-адрес или доменное имя).<br>
- proxy_port (int): Порт прокси-сервера.<br>
- username (str, optional): Имя пользователя для аутентификации на прокси, если требуется. По умолчанию None.<br>
- password (str, optional): Пароль для аутентификации на прокси, если требуется. По умолчанию None.<br>

**Примечания:<br>**
- Если предоставлены имя пользователя и пароль, они будут включены в строку прокси.<br>
- Изменяет поведение модуля socket для использования socks.socksocket.<br>


## Обработчик ошибок (error_handler.py)

**Функция "get_description_and_solution": <br>**

**def get_description_and_solution(error: str):**

**Описание:<br>**
Вызывается когда произошла ошибка при работе с аккаунтами. <br>
Возвращает ошибку и её решение.

**Параметры:<br>**
- error (str): Полученная ошибка 

**Возвращаемое значение:**
- list: возвращает первым параметром списка ошибку(str), вторым её решение(str)


**Функция "error_handler": <br>**

**def error_handler(error: str,id_folder: int,account_type: str):**

**Описание:<br>**
Вызывает функцию get_description_and_solution() для получения описания ошибки и её решения. <br>
Вызывает функции **_change_folder** и **_change_db**, исходя из ошибки передаём туда определённые параметры. <br>
Концом функции будет перемещение аккаунта с ошибкой в необходимую папку, <br>
изменение последовательности id в папке откуда поступил аккаунт и изменение последовательности id БД. <br>
Возможные перемещения в такие папки как: login_error_accounts, temporary_ban_account. <br>
Перемещение в login_error_accounts произойдёт в том случае, если не смогли войти в аккаунт. <br>
Перемещение в temporary_ban_account произойдёт в том случае, если аккаунт получил временный бан. <br>

**Параметры:**<br>
- error (str): Полученная ошибка. <br>
- id_folder: (int): id папки у аккаунта с полученной ошибкой. <br>
- account_type: (str): тип аккаунта у которого получили ошибку.
Возможные принимаемые значения: active, archive, main, login_error, temporary_ban. <br><br>


**Функция "_change_folder": <br>**

**def _change_folder(id_folder: int,account_type: str, target_account_type: str):**

**Описание:<br>**
Копирует указанную папку в указанный тип аккаунтов. <br>
Пример: получили путь 'accounts/active_accounts/3'. 
Этот аккаунт будет перемещён в указанный тип аккаунтов (target_account_type) добавляя его последним id. <br>
Запоминаем этот id и в конце его вернём. <br>
То есть мы получили login_error, и в таком типе аккаунтов последний id это 5, значит аккаунт с ошибкой, 
будет добавлен с id = 6 (accounts/login_error_accounts/6). <br>
Удалит папку откуда скопировали, то есть 'accounts/active_accounts/3'.

Меняет последовательность id в папке с типом аккаунтов откуда получили аккаунт. <br>
Т.к. Удалён аккаунт откуда брали, значит необходимо восстановить последовательность id. 
Меняет id на минус один после папки которую получили из id_folder. <br>
Пример: есть последовательность с удалённым id = 3. Последовательность: 0 1 2 4 5.
Будут переименованы все папки начиная с 4. <br> 
Итог: 0 1 2 3 4

**Параметры:<br>**
- id_folder (int): id папки где произошла ошибка. <br>
- account_type (str): тип аккаунта где произошла ошибка.
Возможные принимаемые значения: active, archive, main, login_error, temporary_ban. <br>
- target_account_type (str): куда перемещать аккаунт.
Возможные принимаемые значения: login_error, temporary_ban.<br>

**Возвращаемое значение:**
- int: возвращает новый id который присвоили в перемещённую папку с необходимым типом аккаунтов.


**Функция "_change_db": <br>**

**def  _change_db(id: int,account_type: str, id_in_target_type: int, target_account_type: str, error_description_solution: list):**

**Описание:<br>**

Меняет последовательность id в БД откуда взяли аккаунт на минус одно значение, всё что идёт после полученного id. <br>

**Параметры:<br>**

### **Класс:** <br>
### show_account(QThread):
Класс `show_account` наследуется от QThread и предназначен для **асинхронного получения информации об аккаунте Telegram.** Используется для: 
- Подключения к аккаунту через tdata папку
- Сбора базовой информации (ID, юзернейм, телефон, геолокация)
- Обработки ошибок подключения
- Работы с прокси (SOCKS5)

При работе с этим классом изначально мы попадём в `run()`, после он уже вызовет асинхронно метод `async get_info_account()`<br>
Это сделано потому что будет возникать ошибка, если сделать метод `run()` асинхронным и вызвать его.

**Конструктор __init__():**
```
def __init__(
    self,
    folder_path_account: str,  # Путь к папке аккаунта (с tdata)
    id_account: int,           # Уникальный ID аккаунта
    account_type: str,          # Тип аккаунта ('active', 'archive' и т.д.)
    use_proxy: bool,            # Флаг использования прокси
    id_proxy: str,              # IP прокси-сервера
    port: int,                  # Порт прокси
    login: str,                 # Логин прокси
    password: str               # Пароль прокси
)
```

**Пример использования:** <br>
Создание потока для аккаунта ID=5 с прокси <br>
```
worker = show_account( 
    folder_path_account = "/accounts/active_accounts/5",
    id_account = 5,
    account_type = "active",
    use_proxy = True,
    id_proxy = "192.168.0.1", 
    port = 1080, 
    login = "proxy_user", 
    password = "qwerty" 
)
```

**Методы:** <br>

**def run(self): <br>**
**Описание:<br>**
Запускает асинхронную задачу get_info_account() через asyncio.run().

**def get_info_account(self): <br>**
**Описание:<br>**
- Настройка прокси (если use_proxy=True): <br>
```
socks.set_default_proxy(socks.SOCKS5, id_proxy, port, True, login, password)
socket.socket = socks.socksocket
```

- Подключение к аккаунту:
  - Использует TDesktop для работы с tdata
  - Конвертирует в Telethon-сессию
  - Таймаут подключения: 7 секунд

- Сбор данных:
  - ID аккаунта (me.id)
  - Юзернейм (me.username)
  - Имя (me.first_name)
  - Телефон (me.phone)
  - Геолокация через [get_country_flag()](#получение-geo-по-номеру-flagpy)

- Обработка ошибок:
  - Гарантирует отключение клиента при ошибках

Возвращаемые типы сигналов: task_done(list, bool) <br>
Первый это список с данными об аккаунте, второй возникновение ошибки на аккаунте (будет True, если есть ошибка)  <br>

### def _show_account_done(self, account_data: list, account_error: bool):
Метод обрабатывает результаты работы потока [show_account](#show_accountqthread), обновляя интерфейс и базу данных. Основные функции:
- Фильтрация успешных/неудачных подключений
- Обновление таблицы аккаунтов
- Обработка ошибок
- Визуальное отображение статуса операции

**Описание:<br>**
Фильтрация входящих данных.
```
if account_error:
    self.error_accounts.append(account_data)  # Добавление в список ошибок
else:
    self.accounts.append(account_data)  # Добавление в список успешных подключений

self.quantity_received_accounts += 1 # т.к. у нас вернулся один аккаунт
```

Проверка завершения обработки, если количество запущенных аккаунтов равно или более количества полученных аккаунтов

```
if self.quantity_received_accounts >= self.quantity_start_accounts:
   socket.socket = self.original_socket  # Восстановление оригинального сокета прокси
```

Сортируем список в порядке возрастания по первому элементу вложенного списка <br>
`sorted_accounts = sorted( self.accounts, key=lambda x: x[0])`

Далее происходит добавление аккаунтов в **tableWidget_account** <br>
Для каждого элемента списка `sorted_accounts` проверяем актуальность данных через метод [self.update_account_data( )](#def-_update_account_dataself-account_data-list) <br>
Получаем доп информацию об аккаунте через БД используя метод [self.get_additional_information( )](#def-_get_additional_informationself-account_data-list---list):
```
for account_info in sorted_accounts:
  self.update_account_data(account_info) # обновление данных в БД 
   for information in self.get_additional_information(account_info): # получение доп информации
       account_info.append(information)
```

Добавляем информацию в **tableWidget_account** <br>
Используя фото для четвёртой колонки ищя по имени ГЕО флаг с таким же названием <br>
Пример: получили гео **RU** значит ищем в папке `resources/pictures_flag/ru.png`, если не находим такой флаг, то используем `default_flag.png` в той же папке.
Используя преобразование в строку для первой колонки. <br>
Разрешаем редактировать только последнею колонку. <br>

Если есть аккаунты с ошибкой `if self.error_accounts:` <br>
Формируем список, который будет хранить другие списки с информацией об ошибке `full_list_error_account = []` <br>

Так же вызываем для каждого элемента списка `self.error_accounts` [error_handler](#обработчик-ошибок-error_handlerpy) для перемещения аккаунтов в которые не смогли войти. <br>

### def _move_accounts(self,button: QPushButton):

**Описание:<br>**
Необходим для перемещения выбранных аккаунтов в полученный тип аккаунтов.

Сперва происходит проверка, если мы находимся в аккаунтах с ошибкой входа, то с таких типов аккаунтов нельзя перемещать. <br>
Тогда выведем соответствующие сообщение. <br>

Получаем список выбранных строк <br>

    - `selected_rows.add(item.row())  # добавляем номер строки и добавляем в множество`  <br>


Исходя из имени полученной кнопки формируем переменную `target_type_account` <br>
- `if button.objectName() == 'pushButton_move_active':` <br>
  - `target_type_account = 'active'` <br>

Формируем две переменные <br> 
`where_from_copy_path` - откуда копировать <br>
`where_copy_path` - куда копировать <br>
на примере куда копировать: <br>
`where_copy_path = self.root_project_dir + f'/accounts/{target_type_account}_accounts' ` <br>
Если переменная `target_type_account` = active, то путь будет следующим: `.../accounts/active_accounts` <br>

Получаем список папок и узнаём id с которого будем вставлять перемещённый аккаунт <br>
- `items = os.listdir(where_copy_path)  # Получаем список файлов и папок в указанной директории` <br>
  - `last_id_in_target_copy = sum(1 for item in items if os.path.isdir(os.path.join(where_copy_path, item)))` <br>
  - `# Фильтруем только папки` <br>

То есть если в папке `.../accounts/active_accounts` находится такие id 0, 1, 2, 3  <br>
Тогда переменная `last_id_in_target_copy` будет иметь значение 4 т.к. именно с такого id будет необходимо вставить аккаунт. <br>
  
Работаем с каждым полученным id <br>
- `for id_folder in selected_rows:` <br>
Копируем из папки с полученным id в папку с необходимым id для вставки <br>
  - `shutil.copytree(where_from_copy_path + f'/{id_folder}',where_copy_path + f'/{last_id_in_target_copy}') # копирование` <br>
  Обновляем данные в БД меняя тип аккаунта на выбранный и меняем id на что присвоенный. <br>
  - `cursor.execute(f"UPDATE accounts SET id = ?, account_status = ?  WHERE id = ? AND account_status = ?",` <br>
  - `(last_id_in_target_copy, target_type_account, id_folder, self.selected_account_type))` <br>
  Удаление что только что скопировали и прибавляя id <br>
  - `shutil.rmtree(where_from_copy_path + f'/{id_folder}')` <br>
  - `last_id_in_target_copy += 1` <br>

Далее необходимо поменять значения id из папки откуда переместили аккаунты <br>
#### Допустим у нас сейчас такая ситуация в папке откуда взяли аккаунты: 0,2,3,7,8,10,14 <br>

    Цикл будет идти до тех пор пока папка НЕ будет найдена (то есть когда необходимой папки не будет) <br>
    - `while True:`  <br>
        - `id_result = id_from_folder_copy` <br>
        Необходимо сразу прибавить + 1 к искомой папке, т.к.  мы будем искать не перемещённую папку а следующею за ней <br>
        - `id_from_folder_copy += 1` <br>
            Ищем папку которую необходимо переименовать на то значение которые мы изъяли из этих папок. <br>
            Возьмём прошлый [пример](#допустим-у-нас-сейчас-такая-ситуация-в-папке-откуда-взяли-аккаунты-023781014-br) 
             нам необходимо поменять папку с id = 2 на id = 1 <br>
            - `if os.path.isdir(where_from_copy_path + f'/{id_from_folder_copy}'):  # если такая папка есть` <br>
                - `os.rename(where_from_copy_path + f'/{id_from_folder_copy}',where_from_copy_path + f'/{id_result}')  # переименование` <br>
                обновляем данные в БД <br>
                - `cursor.execute(f"UPDATE accounts SET id = ?, account_status = ?  WHERE id = ? AND account_status = ?",` <br>
                               `(id_result, self.selected_account_type, id_from_folder_copy, self.selected_account_type))` <br>
                - `connection.commit()  # сохранение изменений` <br>
            - `else:` <br>
                - `break` <br>

Этой логикой обработается такая ситуация когда несколько аккаунтов подряд перемещены. <br>
То есть в такой ситуации 0 1 4 5 6

## Получение geo по номеру (flag.py)
COUNTRY_CODES - Содержит все допустимые двухбуквенные коды стран <br>
Хранит только функцию get_country_flag

### def get_country_flag(phone_number: str) -> str:
Принимает `phone_number: str` - номер телефона <br>
Пример: `+79123456789`  <br>
Возвращает краткое имя страны (str) или `'default_flag'` (str) <br>
Пример: `ru`

Сперва парсим номер телефона: <br>
`parsed_number = phonenumbers.parse(phone_number) ` <br>
получаем код страны (RU) <br>
`country = geocoder.region_code_for_number(parsed_number) # ` <br>
приводим к нижнему регистру и проверяем наличие такой строки в списке `COUNTRY_CODES` <br>
```
if country in COUNTRY_CODES:
    return country
else:
    return "default_flag"
```
